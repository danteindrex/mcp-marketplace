# Requirements Traceability Template

Use one row per atomic requirement.

| Requirement | Domain | Status | Severity | Evidence | Gap Summary |
|---|---|---|---|---|---|
| Short requirement statement | backend/webapp/infra/security/qa | implemented/partial/missing | critical/high/medium/low | absolute file path with line | one sentence |

## Severity Guide

- `critical`: security/protocol correctness or blocking core business flow.
- `high`: major feature incompleteness affecting core user journeys.
- `medium`: partial implementation or meaningful operational risk.
- `low`: polish, non-blocking ergonomics, or documentation debt.

## Evidence Quality

- Prefer source code references first.
- Add docs references only as supporting context.
- Avoid claims without concrete files.
